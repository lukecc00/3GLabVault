# 账号注册与内部邮件地址说明

## 1. 当前能力

当前版本已经支持：

- 用户使用公开注册页提交注册申请
- 系统按姓名拼音自动生成账号
- 系统按 `账号@邮件域名` 自动分配内部邮件地址
- 用户在注册时申请绑定目标群组
- 管理员在后台单个创建成员注册申请
- 管理员按群组批量生成账号并获取临时密码
- 用户使用“账号/内部邮件地址 + 密码”登录
- 首次登录强制改密
- 管理员重置成员密码并重新下发临时密码
- 为后续站内邮件模块保留内部地址标识

## 2. 账号与内部邮件地址生成规则

- 账号字段使用 `username`
- 默认取用户姓名的拼音全拼并转为小写
- 如果账号冲突，会自动追加数字后缀，例如：
  - `zhangsan`
  - `zhangsan2`
  - `zhangsan3`
- 内部邮件地址自动按以下格式生成：

```text
username@MAIL_DOMAIN
```

默认邮件域名来自：

```env
MAIL_DOMAIN=3glab.local
```

例如：

- 姓名：张三
- 自动账号：`zhangsan`
- 自动邮箱：`zhangsan@3glab.local`

## 3. 用户注册流程

用户访问 `/register` 后，需要填写：

- 真实姓名
- 登录密码
- 学号（可选）
- 个人简介（可选）
- 希望绑定的群组（可选，可多选）

提交后系统会：

1. 自动生成拼音账号
2. 自动生成内部邮件地址
3. 保存密码哈希
4. 绑定所选群组
5. 自动赋予 `MEMBER` 角色
6. 将账号状态设为 `PENDING`

说明：

- 新注册用户默认不能直接登录
- 需要管理员在后台审核为 `ACTIVE` 后，才能使用账号或内部邮件地址登录
- 系统不会进行任何线上邮箱开户，也不会创建外部真实邮箱

## 4. 管理员批量生成流程

管理员在 `/admin/users` 页面可以使用批量生成账号功能。

输入格式：

```text
张三,20220001
李四,20220002
王五
```

管理员还可以为这批账号统一绑定一组群组。

系统会为每个成员自动生成：

- 账号
- 内部邮件地址
- 临时密码

并自动完成：

- 账号状态设为 `ACTIVE`
- 绑定指定群组
- 自动赋予 `MEMBER` 角色
- 标记为首次登录必须修改密码

## 5. 登录规则

当前登录页 `/login` 支持两种标识：

- 用户名
- 内部邮件地址

登录方式：

```text
账号/内部邮件地址 + 密码
```

例如：

- `admin`
- `xiyou3g@3glab.local`

都可以作为登录标识。

## 6. 与内部邮件系统的关系

当前版本的地址体系只服务于内部邮件能力和统一登录，不再作为外部邮件系统账号使用：

- 不进行线上邮箱开户
- 不接收外部邮件
- 不向外部发送邮件
- 后续只通过本软件提供内部收件箱、发件箱和按组抄送能力

当前仅需要保留内部地址域名配置：

```env
MAIL_DOMAIN=3glab.local
```

建议使用内部专用域名或仅系统内展示的地址标识，避免与真实公网邮箱产生混淆。

## 7. 密码与安全策略

- 批量生成账号默认返回一次性临时密码
- 初始化管理员账号默认要求首次登录修改密码
- 管理员可在后台对指定成员执行“重置密码并强制改密”
- 登录密码只服务于业务系统本身，不再同步任何外部邮箱密码
