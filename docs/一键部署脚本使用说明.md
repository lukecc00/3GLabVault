# 一键部署脚本使用说明

## 1. 适用场景

适合把整个项目上传到一台已安装 Docker 的 Linux 服务器后，由非专业运维同学直接执行脚本完成初始化和启动。

当前脚本会复用仓库中已有的生产编排：

- `infra/docker/docker-compose.prod.yml`
- `.env.production`
- `scripts/deploy-production.sh`

## 2. 前置条件

部署前请先确认服务器满足以下条件：

- 已安装 Docker
- Docker 支持 `docker compose`
- 服务器已开放 `80` 和 `443` 端口
- 域名已经解析到当前服务器公网 IP

## 3. 最简单的部署方式

进入项目根目录后，直接执行：

```bash
bash scripts/deploy-production.sh
```

或者使用根目录脚本别名：

```bash
pnpm deploy:prod
```

首次执行时，脚本会自动完成以下工作：

1. 交互式生成 `.env.production`
2. 校验生产 Docker Compose 配置
3. 启动 `traefik`、`postgres`、`redis`、`minio`
4. 自动初始化数据库结构和种子数据
5. 构建并启动 `server` 和 `web`
6. 输出当前容器状态和访问地址

## 4. 脚本交互时需要填写的内容

脚本会提示填写以下核心信息：

- 线上域名，例如 `lab.example.com`
- LetsEncrypt 邮箱
- PostgreSQL 用户名和密码
- MinIO 用户名和密码
- JWT 鉴权密钥
- 管理员初始账号信息
- 是否填写 Mailcow API 地址和 Key

其中密码类字段如果不想自己生成，可以直接回车使用脚本给出的默认随机值。

## 5. 常用命令

### 5.1 一键部署 / 更新部署

```bash
bash scripts/deploy-production.sh up
```

或：

```bash
pnpm deploy:prod
```

说明：

- 首次部署会自动初始化数据库
- 后续再次执行会复用现有 `.env.production`
- 如果已经初始化过数据库，脚本默认不会重复执行初始化

### 5.2 重新配置生产环境变量

```bash
bash scripts/deploy-production.sh configure
```

或：

```bash
pnpm deploy:prod:configure
```

说明：

- 会重新生成 `.env.production`
- 如果旧文件存在，脚本会先自动备份一份

### 5.3 查看运行状态

```bash
bash scripts/deploy-production.sh status
```

或：

```bash
pnpm deploy:prod:status
```

### 5.4 查看日志

```bash
bash scripts/deploy-production.sh logs
```

或：

```bash
pnpm deploy:prod:logs
```

### 5.5 停止生产环境

```bash
bash scripts/deploy-production.sh down
```

或：

```bash
pnpm deploy:prod:down
```

### 5.6 重新初始化数据库

```bash
bash scripts/deploy-production.sh reinit
```

或：

```bash
pnpm deploy:prod:reinit
```

适用场景：

- 首次初始化失败
- 你手动清空了数据库卷
- 你明确知道自己要重新写入种子数据

## 6. 生成的关键文件

脚本运行后会生成或使用以下文件：

- `.env.production`
- `.env.production.bak.YYYYMMDDHHMMSS`
- `storage/deploy/prod-initialized`

说明：

- `.env.production` 是生产环境真实配置，请不要提交到仓库
- `prod-initialized` 用于标记数据库已经初始化过，避免重复初始化

## 7. 推荐部署流程

对新手最友好的上线流程如下：

1. 把项目完整上传到服务器
2. 进入项目根目录
3. 执行 `bash scripts/deploy-production.sh`
4. 根据提示填写域名、邮箱、密码等配置
5. 等待容器构建完成
6. 打开浏览器访问脚本输出的 HTTPS 地址
7. 用管理员初始账号登录系统

## 8. 常见问题

### 8.1 为什么证书没有立刻生效？

Traefik 需要通过域名访问并完成 Let’s Encrypt 校验。若域名刚解析完成，证书签发可能会有几十秒延迟。

### 8.2 如果域名还没解析，可以先部署吗？

可以先启动容器，但 HTTPS 和正式外部访问通常要等域名解析完成后才正常。

### 8.3 如何更新代码后重新发布？

拉取或替换新代码后，在项目根目录重新执行：

```bash
bash scripts/deploy-production.sh up
```

脚本会重新构建并启动最新版本的 `web` 和 `server`。
