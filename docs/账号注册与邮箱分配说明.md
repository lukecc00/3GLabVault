# 账号注册与内部邮件地址说明

## 1. 当前能力

当前版本已经支持：

- 用户使用公开注册页提交注册申请
- 系统按姓名拼音自动生成账号
- 系统按 `账号@邮件域名` 自动分配内部邮件地址
- 用户注册时必须填写一个真实外部邮箱作为提醒邮箱
- 用户在注册时申请绑定目标群组
- 管理员在后台单个创建成员注册申请
- 管理员在后台审核时可补录或修改成员的真实外部邮箱
- 管理员按群组批量生成账号并获取临时密码
- 用户使用“账号/内部邮件地址 + 密码”登录
- 首次登录强制改密
- 管理员重置成员密码并重新下发临时密码
- 为后续站内邮件模块保留内部地址标识
- 内部邮件到达时，可通过外部 SMTP 向提醒邮箱发送“仅提醒、不含正文”的通知邮件

## 2. 账号与内部邮件地址生成规则

- 账号字段使用 `username`
- 默认取用户姓名的拼音全拼并转为小写
- 如果账号冲突，会自动追加数字后缀，例如：
  - `zhangsan`
  - `zhangsan2`
  - `zhangsan3`
- 内部邮件地址自动按以下格式生成：

```text
username@MAIL_DOMAIN
```

默认邮件域名来自：

```env
MAIL_DOMAIN=3glab
```

例如：

- 姓名：张三
- 自动账号：`zhangsan`
- 自动邮箱：`zhangsan@3glab`

## 3. 用户注册流程

用户访问 `/register` 后，需要填写：

- 真实姓名
- 姓名拼音
- 登录密码
- 真实外部邮箱（必填，用于接收站内邮件提醒）
- 个人简介（可选）
- 希望绑定的群组（可选，可多选）

提交后系统会：

1. 自动生成拼音账号
2. 自动生成内部邮件地址
3. 保存密码哈希
4. 绑定所选群组
5. 自动赋予 `MEMBER` 角色
6. 将账号状态设为 `PENDING`
7. 保存外部提醒邮箱

说明：

- 新注册用户默认不能直接登录
- 需要管理员在后台审核为 `ACTIVE` 后，才能使用账号或内部邮件地址登录
- 系统不会创建外部真实邮箱
- 外部邮箱只用于接收提醒通知，不参与系统登录，也不保存站内邮件正文

## 4. 外部提醒邮箱使用说明

### 4.1 作用

外部提醒邮箱只用于以下场景：

- 当用户收到新的内部邮件时，系统向该外部邮箱发送一封提醒邮件
- 只有主送收件人（`To`）会收到外部提醒；抄送收件人（`Cc`）不会收到中转邮箱提醒
- 提醒邮件只提示“你有新的内部邮件待处理”
- 提醒邮件不包含内部邮件正文、附件、群组信息和完整收件人列表

### 4.2 频控规则

- 同一用户在 1 分钟内最多收到 1 封外部提醒邮件
- 如果在 1 分钟内连续收到多封内部邮件，系统只发送 1 次提醒

### 4.3 管理规则

- 用户公开注册时必须填写外部提醒邮箱
- 管理员在 `/admin/users` 的“新成员注册”中必须填写外部提醒邮箱
- 管理员在 `/admin/users` 的“审核与分配”区域可以补录或修改已选用户的外部提醒邮箱
- 管理员批量生成账号时需要同时录入外部提醒邮箱，生成后即可启用内部邮件提醒

## 5. 管理员批量生成流程

管理员在 `/admin/users` 页面可以使用批量生成账号功能。

输入格式：

```text
张三,zhangsan@example.com
李四,lisi@example.com
```

管理员必须为这批账号统一设置一组群组和统一初始密码。

系统会为每个成员自动生成：

- 账号
- 内部邮件地址
- 外部提醒邮箱绑定
- 临时密码（使用本次设置的统一初始密码）

账号生成规则：

- 默认根据姓名自动生成小写拼音账号
- 若拼音账号已存在，则会在拼音后随机追加 2 位数字重新尝试生成唯一账号
- 成员首次使用初始密码登录后，必须立即修改密码；修改完成后需使用新密码重新登录

并自动完成：

- 账号状态设为 `ACTIVE`
- 绑定指定群组
- 自动赋予 `MEMBER` 角色
- 标记为首次登录必须修改密码

## 6. 登录规则

当前登录页 `/login` 支持两种标识：

- 用户名
- 内部邮件地址

登录方式：

```text
账号/内部邮件地址 + 密码
```

例如：

- `admin`
- `xiyou3g@3glab`

都可以作为登录标识。

## 7. 与内部邮件系统的关系

当前版本的地址体系只服务于内部邮件能力和统一登录，不再作为外部邮件系统账号使用：

- 不进行线上邮箱开户
- 不接收外部邮件
- 不向外部发送内部邮件正文
- 后续只通过本软件提供内部收件箱、发件箱和按组抄送能力

补充说明：

- 系统允许通过单独配置的 SMTP 账号向用户的外部提醒邮箱发送“仅提醒、不含正文”的通知
- 该提醒链路不改变“内部邮件仅站内存储”的边界
- 外部提醒失败不会影响内部邮件在系统内的正常投递

当前仅需要保留内部地址域名配置：

```env
MAIL_DOMAIN=3glab
```

建议使用内部专用域名或仅系统内展示的地址标识，避免与真实公网邮箱产生混淆。

## 8. 密码与安全策略

- 批量生成账号默认返回一次性临时密码
- 初始化管理员账号默认要求首次登录修改密码
- 管理员可在后台对指定成员执行“重置密码并强制改密”
- 登录密码只服务于业务系统本身，不再同步任何外部邮箱密码
- 外部提醒邮箱不参与系统鉴权，不能作为登录标识

## 9. QQ SMTP 提醒配置示例

如果使用 QQ 邮箱作为提醒中转账号，可在后端环境变量中配置：

```env
EXTERNAL_MAIL_REMINDER_ENABLED=true
SMTP_HOST=smtp.qq.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=1793026645@qq.com
SMTP_PASS=你的QQ邮箱SMTP授权码
SMTP_FROM=1793026645@qq.com
APP_BASE_URL=https://lab.example.com
```

字段说明：

- `EXTERNAL_MAIL_REMINDER_ENABLED`
  - 含义：是否启用“外部提醒邮箱”能力
  - 用法：`true` 表示启用，`false` 表示关闭
  - 影响：关闭后，系统内部邮件仍可正常使用，但不会向外部邮箱发送提醒
- `SMTP_HOST`
  - 含义：SMTP 发信服务器地址
  - 用法：QQ 邮箱固定填写 `smtp.qq.com`
  - 说明：切换到其他邮箱服务商时，需要改成该服务商的 SMTP 主机地址
- `SMTP_PORT`
  - 含义：SMTP 发信端口
  - 用法：QQ 邮箱常用 `465`
  - 说明：若切换服务商，请按该服务商文档调整端口
- `SMTP_SECURE`
  - 含义：是否使用加密连接
  - 用法：QQ 邮箱配合 `465` 时填写 `true`
  - 说明：若服务商要求明文端口或 STARTTLS，需要按文档调整
- `SMTP_USER`
  - 含义：系统用于登录 SMTP 的发信账号
  - 用法：填写开启了 SMTP 服务的 QQ 邮箱地址
  - 说明：系统会使用这个账号发出提醒邮件
- `SMTP_PASS`
  - 含义：SMTP 登录授权码
  - 用法：填写 QQ 邮箱后台生成的 SMTP 授权码
  - 说明：不能填写 QQ 登录密码
- `SMTP_FROM`
  - 含义：提醒邮件显示出来的发件人地址
  - 用法：通常与 `SMTP_USER` 保持一致
  - 说明：若服务商限制发件人与登录账号一致，则必须填写同一个邮箱
- `APP_BASE_URL`
  - 含义：系统对外访问地址
  - 用法：填写实际站点地址，例如 `https://lab.example.com`
  - 说明：提醒邮件里的“登录入口”链接由它生成

使用说明：

- `SMTP_PASS` 必须填写 QQ 邮箱开启 SMTP 后生成的授权码，不能填写 QQ 登录密码
- `SMTP_FROM` 建议与 `SMTP_USER` 保持一致
- `APP_BASE_URL` 用于生成提醒邮件中的系统登录入口链接
- 如果未配置 SMTP，系统内部邮件仍可正常使用，只是不会发送外部提醒

切换提醒邮箱账号的方法：

1. 用新的 QQ 邮箱登录网页版邮箱后台
2. 为新账号开启 SMTP 服务
3. 生成新的 SMTP 授权码
4. 修改后端环境变量中的以下字段：

```env
SMTP_USER=新的QQ邮箱地址
SMTP_PASS=新的QQ邮箱SMTP授权码
SMTP_FROM=新的QQ邮箱地址
```

5. 如有需要，同时确认 `SMTP_HOST`、`SMTP_PORT`、`SMTP_SECURE` 仍与新服务商配置一致
6. 重启后端服务，使新配置生效
7. 用测试用户发送一封内部邮件，确认新账号可以正常发出提醒

切换账号时的注意事项：

- 旧账号的授权码不能复用于新账号
- 如果新账号未开启 SMTP，即使邮箱密码正确也无法发送提醒
- 建议切换完成后废弃旧账号授权码，避免遗留风险
- 若更换为非 QQ 邮箱服务，还需要同步更新 `SMTP_HOST`、`SMTP_PORT`、`SMTP_SECURE`
