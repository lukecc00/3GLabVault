# 生产访问限制说明

## 目标

- 默认仅允许中国地区访问站点
- 在反向代理 / CDN / WAF 层优先拦截海外请求
- 应用层保留国家码白名单兜底

## 推荐顺序

1. CDN / WAF 按国家地区限制，仅放行 `CN`
2. Nginx / Traefik 继续校验国家码请求头
3. 应用层通过 `ACCESS_ALLOWED_COUNTRIES` 再做一次兜底

## 应用层配置

服务端已支持以下环境变量：

- `ACCESS_ALLOWED_COUNTRIES=CN`
- `ACCESS_COUNTRY_HEADER=cf-ipcountry`
- `ACCESS_COUNTRY_STRICT=false`

说明：

- `ACCESS_ALLOWED_COUNTRIES` 支持逗号分隔多个国家码
- `ACCESS_COUNTRY_HEADER` 应与上游代理透传的国家码请求头一致
- `ACCESS_COUNTRY_STRICT=true` 时，缺少国家码的请求也会被拦截

## Nginx 示例

可参考 `infra/docker/nginx/3glabvault.conf.example`：

- 使用 `CF-IPCountry` 作为国家码来源
- 非 `CN` 请求直接返回 `403`
- 同时把国家码继续透传给后端

## 注意事项

- 仅靠应用层无法替代专业 WAF / CDN 国家访问控制
- 如果上游没有注入可信国家码请求头，应用层国家限制不会可靠
- 内网探活、容器间访问、本机回环访问默认不受国家限制影响
