# 安全审计与回查说明

## 目标

本项目已增加针对敏感安全事件的审计日志能力，目标是做到：

- 能回查是谁发起了敏感操作
- 能回查操作发生在什么时间
- 能回查请求来自哪个 IP、哪个国家地区
- 能回查操作作用到了哪个账号、页面、图片或邮件
- 能结合当前工作身份判断操作时的权限上下文

## 审计数据来源

后端通过以下能力生成审计数据：

- `RequestContextMiddleware`
  - 统一采集 `requestId`
  - 统一采集客户端 IP
  - 统一采集国家码请求头
  - 统一采集 `User-Agent`
  - 统一采集 `X-Active-Workspace`

- `AuditLogService`
  - 将敏感操作写入 `AuditLog` 表
  - 写入失败不会中断主业务，但会输出服务端错误日志

## 后台查询页

系统管理员与具备全局后台权限的管理员现在可直接在后台访问：

- `/admin/audit-logs`

该页面提供以下能力：

- 默认按时间倒序查看最新高危操作
- 按操作用户筛选
- 按来源 IP 筛选
- 按国家码筛选
- 按动作关键词筛选
- 按资源类型与资源关键词筛选
- 按时间范围筛选
- 直接查看操作用户、资源、请求来源与发生时间

因此，常规排查已不再需要优先直接查数据库日志，建议先使用后台查询页完成第一轮回查。

## AuditLog 表主要字段

- `actorId`
  - 发起操作的用户 ID
  - 匿名或无法识别操作者时可为空

- `action`
  - 操作类型
  - 例如登录、改密、授权、发信、删信、图片访问

- `targetType`
  - 被操作资源类型
  - 例如 `USER`、`KNOWLEDGE_PAGE`、`KNOWLEDGE_ASSET`、`INTERNAL_MAIL_MESSAGE`

- `targetId`
  - 被操作资源 ID 或对象键

- `status`
  - 操作结果
  - 当前支持 `SUCCESS`、`FAILURE`、`DENIED`

- `summary`
  - 便于人工快速理解的简短中文描述

- `metadata`
  - 结构化补充信息
  - 例如角色列表、群组列表、收件人数、图片尺寸、删除数量

- `ipAddress`
  - 请求来源 IP

- `countryCode`
  - 来源国家码

- `userAgent`
  - 客户端标识

- `workspaceId`
  - 当次操作使用的工作身份

- `createdAt`
  - 审计事件发生时间

## 当前已接入的审计范围

- 认证安全
  - 登录成功
  - 登录失败
  - 修改密码成功
  - 修改密码失败

- 用户与管理员敏感操作
  - 批量生成账号
  - 审核用户
  - 管理员重置密码
  - 更新用户角色
  - 更新用户群组
  - 归档用户
  - 恢复归档内容
  - 重新启用用户

- 知识库安全
  - 上传知识库图片
  - 访问知识库图片资源
  - 创建知识页
  - 更新知识页
  - 删除知识页
  - 恢复知识页
  - 授予知识页编辑权限
  - 移除知识页编辑权限

- 内部邮件安全
  - 查看邮件详情
  - 发送邮件
  - 保存草稿
  - 更新邮箱状态
  - 批量删除邮件
  - 彻底删除邮件
  - 清空回收站

## 回查思路

### 1. 按人回查

适合排查：

- 某管理员是否重置过某账号密码
- 某成员是否访问过某知识库资源
- 某人是否批量删除过邮件

建议关注字段：

- `actorId`
- `action`
- `targetType`
- `targetId`
- `createdAt`
- `ipAddress`
- `workspaceId`

### 2. 按资源回查

适合排查：

- 某知识页是谁删的
- 某图片资源被谁访问过
- 某封邮件被谁彻底删除过

建议关注字段：

- `targetType`
- `targetId`
- `action`
- `actorId`
- `createdAt`

### 3. 按来源 IP / 国家回查

适合排查：

- 是否存在异常地区访问
- 是否存在同一海外来源持续撞库
- 是否存在不同账号共享同一异常来源

建议关注字段：

- `ipAddress`
- `countryCode`
- `action`
- `status`
- `createdAt`

### 4. 按失败事件回查

适合排查：

- 登录爆破
- 恶意改密尝试
- 权限探测

建议关注字段：

- `status = FAILURE` 或 `DENIED`
- `action`
- `ipAddress`
- `countryCode`
- `createdAt`

## 推荐查询方式

如果直接查数据库，优先按以下维度组合筛选：

- 时间范围 + `action`
- `actorId` + 时间范围
- `targetType` + `targetId`
- `ipAddress` + 时间范围
- `countryCode` + `status`

如果后续接入 Grafana / Loki / ELK，建议把 `AuditLog` 同步到日志检索系统，并按以下维度建立看板：

- 登录失败次数
- 海外访问拦截次数
- 管理员重置密码次数
- 知识库图片访问热点
- 邮件删除和彻底删除次数

## 运维建议

- 生产环境必须保证国家码请求头来自可信代理，不要相信客户端自带同名请求头
- 建议在 CDN / WAF 层先按国家限制，仅把可信请求转发到应用层
- 建议为 `AuditLog` 建立定期归档策略，避免数据长期无限增长
- 建议后续增加后台审计查询页面，避免每次都直接查库
- 建议为高频失败登录、集中海外访问、管理员高危操作增加告警
