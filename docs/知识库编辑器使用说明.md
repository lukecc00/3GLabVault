# 知识库编辑器使用说明

本文档说明当前知识库编辑器的使用方式、三种编辑模式、常用工具栏能力、快捷键和注意事项。

## 1. 编辑器模式

当前知识库编辑器提供 3 种模式：

### 1.1 可视化编辑

- 适合日常写作。
- 正文区域直接按最终阅读样式编辑，接近飞书云文档。
- 支持标题、列表、引用、代码、图片、颜色等能力。

### 1.2 分栏编辑

- 左侧是 Markdown 源码。
- 右侧是实时渲染结构。
- 适合需要手写 Markdown 的场景，例如快速调整表格、任务列表、引用块和代码块。
- 工具栏操作和快捷键同样支持在左侧源码区生效。

### 1.3 阅读预览

- 仅展示最终阅读态。
- 适合发布前检查标题层级、代码高亮、图片显示和整体排版。

## 2. 工具栏能力

编辑器当前支持以下工具栏功能：

- 加粗
- 斜体
- 下划线
- 行内代码
- 链接
- 引用
- 无序列表
- 有序列表
- H1
- H2
- H3
- 代码块
- 图片链接
- 本地图片
- 文字颜色
- 待办清单
- 对比表格
- 提示块
- 分隔线
- 撤销
- 重做
- 清除格式

补充说明：

- 鼠标悬停在工具栏功能按钮上，会直接显示该按钮对应的快捷键。
- 点击编辑器右上角的 `?` 按钮，或按 `Cmd/Ctrl + /`，可查看完整快捷键说明。

## 3. 颜色功能

颜色功能同时提供两种入口：

- 常驻色块：工具栏里始终可见，可一键应用颜色。
- 浮层面板：点击 `文字颜色` 按钮后展开，更接近飞书的交互方式。

当前支持颜色：

- 默认
- 蓝色
- 绿色
- 黄色
- 红色
- 紫色

说明：

- 可视化编辑模式下，颜色会直接应用到选中文本。
- 文本在保持选中状态时，颜色变化也会立即可见，不需要先取消选中。
- 分栏模式下，颜色会写入 Markdown 源码中的 HTML `span` 标记。
- 渲染器仅允许当前定义的几种安全颜色，避免任意样式污染正文。

## 4. 图片使用

### 4.1 图片链接

- 点击 `图片链接`
- 输入图片地址
- 输入图片说明
- 插入后会在正文中显示图片

### 4.2 本地图片

- 点击 `本地图片`
- 选择本地图片文件
- 当前会以内联方式插入正文

注意：

- 单张图片建议控制在 3MB 以内。
- 大图过多会影响页面体积与加载速度。

## 5. 快捷键

### 5.1 飞书一致的快捷键

- 查看快捷键：`Cmd/Ctrl + /`
- 加粗：`Cmd/Ctrl + B`
- 斜体：`Cmd/Ctrl + I`
- 下划线：`Cmd/Ctrl + U`
- 链接：`Cmd/Ctrl + K`
- 行内代码：
  - Windows：`Ctrl + Shift + C`
  - macOS：`Ctrl + Cmd + C`
- 文字颜色：
  - Windows：`Ctrl + Shift + E`
  - macOS：`Cmd + Shift + E`
- H1：
  - 通用简写：`Alt + 1`
  - 兼容组合：`Cmd/Ctrl + Alt + 1`
- H2：
  - 通用简写：`Alt + 2`
  - 兼容组合：`Cmd/Ctrl + Alt + 2`
- H3：
  - 通用简写：`Alt + 3`
  - 兼容组合：`Cmd/Ctrl + Alt + 3`
- 无序列表：`Cmd/Ctrl + Shift + 8`
- 有序列表：`Cmd/Ctrl + Shift + 7`
- 引用：`Cmd/Ctrl + Shift + .`
- 代码块：
  - Windows：`Ctrl + Alt + C`
  - macOS：`Cmd + Option + C`
- 待办清单：
  - Windows：`Ctrl + Alt + T`
  - macOS：`Cmd + Option + T`
- 分隔线：
  - Windows：`Ctrl + Alt + S`
  - macOS：`Cmd + Option + S`
- 撤销：`Cmd/Ctrl + Z`
- 重做：`Shift + Cmd/Ctrl + Z`

### 5.2 项目扩展快捷键

以下功能不是飞书官方标准直达键，属于本项目补充：

- 图片链接：`Cmd/Ctrl + Shift + K`
- 本地图片：`Cmd/Ctrl + Shift + I`
- 对比表格：`Cmd/Ctrl + Shift + T`
- 提示块：`Cmd/Ctrl + Shift + H`

### 5.3 分栏 Markdown 直输语法

在分栏编辑模式左侧 Markdown 源码区，也可以直接输入以下语法：

- H1：`# + 空格`
- H2：`## + 空格`
- H3：`### + 空格`
- 无序列表：`- + 空格` 或 `* + 空格`
- 有序列表：`1. + 空格`
- 待办清单：`[] + 空格`
- 引用：`> + 空格`
- 行内代码：`` `文本` ``
- 代码块：`` ``` + 空格 ``
- 分隔线：`---` 或 `***`

## 6. mac 键盘中的 Alt 是什么

在 mac 上：

- 键盘上通常显示的是 `Option` 或 `⌥`
- 浏览器事件中对应的是 `Alt`
- 所以代码里判断 `altKey` 是正确的

如果文档里写：

- `Alt`
- `Alt/Option`
- `Option`

在 mac 环境中，本质上指的都是同一个按键。

建议理解方式：

- Windows 用户按 `Alt`
- mac 用户按 `Option (⌥)`

## 7. 使用建议

- 日常编辑优先使用 `可视化编辑`
- 需要精确控制 Markdown 时切换到 `分栏编辑`
- 发布前使用 `阅读预览` 检查最终效果
- 插入大量图片前，优先压缩图片体积
- 颜色只用于重点强调，不建议整段大面积着色

## 8. 常见问题

### 8.1 为什么分栏模式左边不是可视化正文？

这是刻意设计：

- `可视化编辑` 已负责所见即所得写作
- `分栏编辑` 专门用于 `Markdown 源码 + 渲染结构` 对照

### 8.2 为什么颜色有两种入口？

为了兼顾两类使用习惯：

- 常驻色块适合高频快速操作
- 浮层面板更接近飞书，也更适合集中查看颜色选项

### 8.3 快捷键在哪查看？

有两种方式：

- 点击编辑器顶部的 `?` 按钮
- 直接按 `Cmd/Ctrl + /`

### 8.4 为什么鼠标悬停按钮会显示快捷键？

这是编辑器的辅助提示能力，作用是：

- 降低记忆成本
- 让新成员快速熟悉工具栏
- 在不打开完整快捷键面板的情况下，直接查看当前按钮的快捷键
